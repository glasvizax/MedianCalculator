\# Тестовое задание для C++ разработчика

\#\# Введение

Данное тестовое задание предназначено для оценки кандидатов на позицию C++ разработчика.

Задание проверяет навыки работы с современным C++23, обработкой данных, файловой системой и библиотеками Boost.

\*\*Цель задания:\*\* Разработать консольное приложение для расчета медианы цен из CSV-файлов с потоковыми данными биржевых торгов.

\---  
\#\# 1\. Описание задачи

\#\#\# 1.1. Общая задача

Вам предстоит создать консольное приложение для анализа биржевых данных. Приложение читает CSV-файлы с историей торгов из указанной директории, объединяет данные из всех подходящих файлов в единый поток, вычисляет медиану цен по мере обработки данных и сохраняет результаты в единственный выходной файл.

Суть задачи: По мере чтения каждой новой цены из объединённого потока данных, программа пересчитывает медиану с учетом всех ранее прочитанных значений. Когда медиана изменяется — приложение фиксирует это в результирующем файле вместе с временной меткой.

Ключевая особенность: Медиана рассчитывается постепенно, добавляя по одному значению. Например: прочитали цену 100 → медиана 100; добавили цену 102 → медиана 101; добавили цену 99 → медиана 100\. Каждое новое значение учитывает все предыдущие, а не вычисляется заново по всему набору данных.

\#\#\# 1.2. Входные данные

Приложение читает CSV-файлы из указанной директории. Каждый файл представляет определенный тип биржевых данных. Данные из всех файлов, прошедших фильтр \`filename\_mask\`, объединяются в единый набор и сортируются по \`receive\_ts\`. Медиана рассчитывается по полю \`price\` из всех файлов, а не из какого-то одного.

\> В рамках данного задания необходимо самостоятельно сгенерировать входные данные, например с помощью LLM

\#\#\#\# Структура файла \`level.csv\`:

\> \*\*Примечание:\*\* Разделитель колонок — точка с запятой (\`;\`).

\`\`\`csv  
receive\_ts;exchange\_ts;price;quantity;side;rebuild  
1716810808593627;1716810808574000;68480.00000000;10.10900000;bid;1  
1716810808593627;1716810808574000;68479.90000000;0.00400000;bid;0  
1716810808593627;1716810808574000;68480.20000000;4.52800000;ask;0  
1716810808593627;1716810808574000;68480.40000000;0.10200000;ask;0  
\`\`\`

\*\*Описание полей:\*\*  
\- \`receive\_ts\` — временная метка получения данных (микросекунды)  
\- \`exchange\_ts\` — временная метка биржи (микросекунды)  
\- \`price\` — цена  
\- \`quantity\` — объем  
\- \`side\` — сторона (\`bid\` или \`ask\`)  
\- \`rebuild\` — флаг перестроения стакана (0 или 1\)

\#\#\#\# Структура файла \`trade.csv\`:

\`\`\`csv  
receive\_ts;exchange\_ts;price;quantity;side  
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid  
1716810809314641;1716810809312000;68480.00000000;0.00200000;ask  
1716810809719209;1716810809717000;68480.10000000;0.00500000;bid  
1716810809719209;1716810809717000;68480.10000000;0.00200000;bid  
1716810809719209;1716810809717000;68480.10000000;0.16900000;bid  
\`\`\`

\*\*Описание полей:\*\*  
\- \`receive\_ts\` — временная метка получения данных (микросекунды)  
\- \`exchange\_ts\` — временная метка биржи (микросекунды)  
\- \`price\` — цена сделки  
\- \`quantity\` — объем сделки  
\- \`side\` — сторона (\`bid\` или \`ask\`)

\#\#\# 1.3. Выходные данные

Приложение создает один CSV-файл с фиксированным именем \`median\_result.csv\` в выходной директории. При повторном запуске файл перезаписывается.

Формат файла — две колонки, значение медианы выводится с точностью \*\*8 знаков после запятой\*\*:

\`\`\`csv  
receive\_ts;price\_median  
1716810808663260;68480.10000000  
1716810809314641;68480.05000000  
1716810809719209;68480.10000000  
\`\`\`

\*\*Описание полей:\*\*  
\- \`receive\_ts\` — временная метка события, которое изменило медиану  
\- \`price\_median\` — значение медианы, рассчитанное на основе всех цен, считанных до этого момента

\*\*Важно:\*\*  
\- В выходной файл записывается строка \*\*только при изменении значения медианы\*\*. Изменением считается любое отклонение с точностью до 8 знаков после запятой.  
\- Дублирование \`receive\_ts\` в выходном файле допустимо: если несколько записей с одинаковым \`receive\_ts\` последовательно изменяют медиану, каждое изменение фиксируется отдельной строкой.

\#\# 2\. Конфигурация приложения

\#\#\# 2.1. Параметры командной строки

Приложение принимает путь к конфигурационному файлу через следующие аргументы:

\`\`\`bash  
./csv\_median\_calculator \-config /path/to/config.toml  
*\# или*  
./csv\_median\_calculator \-cfg /path/to/config.toml  
\`\`\`

\*\*Поддерживаемые аргументы:\*\* \`-cfg\`, \`-config\`

Если аргумент не указан, приложение ищет файл \`config.toml\` в той же директории, где находится исполняемый файл.

\#\#\# 2.2. Формат конфигурационного файла

Конфигурация задается в формате TOML:

\`\`\`toml  
\[main\]  
    \# Путь к директории с входными CSV-файлами  
    input \= '/path/to/input\_dir'  
     
    \# Путь к директории для выходных CSV-файлов  
    \# Если не указан, создается директория 'output' рядом с приложением  
    output \= '/path/to/output\_dir'  
     
    \# Список масок имен файлов для фильтрации  
    \# Если список пустой, читаются все CSV-файлы из директории  
    filename\_mask \= \['level', 'trade'\]  
\`\`\`

\*\*Параметры:\*\*  
\- \`input\` (обязательный) — путь к директории с входными CSV-файлами  
\- \`output\` (опциональный) — путь к директории для выходных файлов. Если не указан, создается директория \`output\` в текущей рабочей директории  
\- \`filename\_mask\` (опциональный) — список строк для фильтрации имен файлов. Фильтрация работает как \*\*поиск подстроки\*\*: читаются только те файлы, имена которых содержат хотя бы одну из указанных масок. Например, маска \`'level'\` соответствует файлу \`btcusdt\_level\_2024.csv\`, так как имя файла содержит подстроку \`level\`. Если список пустой, читаются все CSV-файлы

\#\# 3\. Алгоритм работы приложения

\#\#\# 3.1. Общий алгоритм

1\. \*\*Парсинг аргументов командной строки\*\* с использованием \`Boost.Program\_options\`  
2\. \*\*Чтение и валидация конфигурационного файла\*\* (формат TOML)  
3\. \*\*Сканирование файловой системы\*\*: поиск CSV-файлов в указанной директории с применением фильтров из \`filename\_mask\`. Найденные файлы \*\*сортируются лексикографически по имени\*\* перед дальнейшей обработкой  
4\. \*\*Чтение и объединение данных\*\*: считывание всех подходящих CSV-файлов \*\*в лексикографическом порядке имён\*\* и последовательная конкатенация записей в единый набор. Если файл имеет некорректный формат CSV (невалидный заголовок, повреждённые данные, отсутствующие обязательные колонки), файл целиком исключается из обработки с логированием ошибки — источник данных считается невалидным  
5\. \*\*Сортировка данных\*\* по полю \`receive\_ts\` с использованием \`std::stable\_sort\` для сохранения относительного порядка записей с одинаковым \`receive\_ts\`. Поскольку файлы объединяются в лексикографическом порядке, при равном \`receive\_ts\` записи из файла с лексикографически меньшим именем будут идти первыми — это обеспечивает детерминированный и воспроизводимый результат  
6\. \*\*Инкрементальный расчет медианы\*\*:  
   \- Поочередная обработка каждой записи из объединённого отсортированного потока  
   \- Добавление цены в структуру данных для расчета медианы  
   \- Вычисление текущей медианы  
   \- Запись результата при изменении медианы (с точностью до 8 знаков после запятой)  
7\. \*\*Запись результатов\*\* в выходной CSV-файл \`median\_result.csv\`

\#\#\# 3.2. Расчет медианы

\> \*\*Медиана\*\* — это значение, которое делит упорядоченный набор данных пополам. Для четного количества элементов медиана рассчитывается как среднее арифметическое двух центральных значений.

Основная задача приложения — расчет медианы цены в каждый момент времени на основе всех ранее считанных значений цены из всех обрабатываемых файлов.

\*\*Ключевое требование:\*\* Медиана должна рассчитываться \*\*инкрементально\*\*. Выбор способа реализации — на усмотрение кандидата: допускается использование \`Boost.Accumulators\` (с точной медианой) или собственная реализация (например, на основе двух heap'ов).

На каждом шаге обработки:  
\- Добавляется новое значение цены из текущей записи  
\- Вычисляется медиана на основе \*\*всех\*\* значений, обработанных с начала работы  
\- Если медиана изменилась по сравнению с предыдущим значением (с точностью до 8 знаков после запятой), записывается новая строка в выходной файл

\*\*Пример:\*\*

\`\`\`  
Шаг 1: price=68480.10, медиана=68480.10000000 → запись в файл  
Шаг 2: price=68480.00, медиана=68480.05000000 → запись в файл    
Шаг 3: price=68480.10, медиана=68480.10000000 → запись в файл  
Шаг 4: price=68480.10, медиана=68480.10000000 → медиана не изменилась, запись НЕ производится  
\`\`\`

\#\#\# Пример 4: Обработка данных с несколькими временными метками

\*\*Входные данные (\`trade.csv\`):\*\*  
\`\`\`csv  
receive\_ts;exchange\_ts;price;quantity;side  
1000;900;100.00000000;1.00000000;bid  
2000;1900;101.00000000;2.00000000;ask  
2000;1900;102.00000000;1.50000000;bid  
3000;2900;103.00000000;3.00000000;ask  
\`\`\`

\*\*Расчет медианы:\*\*

1\. \`receive\_ts \= 1000\`, \`price \= 100.0\`  
   \- Значения: \`\[100.0\]\`  
   \- Медиана: \`100.00000000\`

2\. \`receive\_ts \= 2000\`, \`price \= 101.0\`  
   \- Значения: \`\[100.0, 101.0\]\`  
   \- Медиана: \`(100.0 \+ 101.0) / 2 \= 100.50000000\`

3\. \`receive\_ts \= 2000\`, \`price \= 102.0\`  
   \- Значения: \`\[100.0, 101.0, 102.0\]\`  
   \- Медиана: \`101.00000000\`

4\. \`receive\_ts \= 3000\`, \`price \= 103.0\`  
   \- Значения: \`\[100.0, 101.0, 102.0, 103.0\]\`  
   \- Медиана: \`(101.0 \+ 102.0) / 2 \= 101.50000000\`

\*\*Выходной файл (\`median\_result.csv\`):\*\*  
\`\`\`csv  
receive\_ts;price\_median  
1000;100.00000000  
2000;100.50000000  
2000;101.00000000  
3000;101.50000000  
\`\`\`

\> Записи с \`receive\_ts \= 2000\` дублируются в выходном файле, так как каждая из них изменила медиану. При использовании \`std::stable\_sort\` относительный порядок записей с одинаковым \`receive\_ts\` сохраняется.
